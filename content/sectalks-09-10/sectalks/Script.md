## **Part 2: The Core Problem - Aggregation Risk**

### **Why These Platforms Are High-Value Targets**

```
Traditional breach:     Single system → Limited damage
Workflow breach:        Single system → ALL connected systems
```

**What's at stake:**

- Credentials for AWS, GitHub, databases, SaaS platforms
- Proprietary business logic and workflows
- API keys and access tokens
- Customer data in workflows
- AI prompts and training data

**Impact multiplication**: One compromise = entire infrastructure exposed

Self-Host Vs SaaS

Show both checklists side-by-side. Highlight:

1. **What disappears** (infrastructure - huge!)
2. **What stays the same** (credentials, workflows, access - most things!)
3. **What's new** (vendor management)
4. **Net result**: Real time savings, but not a security silver bullet


Cloud self hosted


**Key Message for SecTalks**:

1. **Cloud self-hosted ≠ "Easy"** - you still own 80% of security work
2. **Economics favor SaaS below ~50 users**, self-hosted above
3. **Effort reduction with SaaS: ~55%** - significant but not 90%
4. **The 180-item checklist doesn't lie** - look at what you're actually signing up for
5. **Most teams underestimate ongoing maintenance** - it's real work forever

Show the checklist. Let them see the 180 items they need to do for cloud self-hosted vs 100 for SaaS. That visual tells the story better than anything else.
